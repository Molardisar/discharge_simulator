## [1.1.0] - 2026-04-17

### ✨ 新增功能

- **多段工况支持**
  - 动态工况段管理 UI（默认 1 段，支持多段）
  - "+ 添加工况段" / "🗑️删除" 按钮
  - 自动显示总时长、段数
  - 链式模拟：上一段结束 SoC → 下一段初始 SoC
  - 图表标注工况边界（虚线分隔）

- **核心库重构**
  - 新增 `simulator_core.py`：可复用算法库
  - `simulate_multi_segment_power()`: 多段工况模拟引擎
  - 支持提前终止（电压<2.5V 或电量耗尽）

### 🔧 技术实现

- **向后兼容**
  - 1 段工况 = 原有单段模式
  - 现有用户无感知升级
  - UI 默认显示 1 段，渐进复杂

- **智能初始电流**
  - 每段使用 `P/3.7V` 估算初始电流
  - 避免硬编码值

### 📊 使用示例

**单段模式**（默认）：
```
工况 #1: 290W, 90s
→ 模拟 90 秒恒功率放电
```

**多段模式**：
```
工况 #1: 290W, 90s
工况 #2: 150W, 60s
工况 #3: 400W, 30s
→ 链式模拟 180 秒完整 cycle
```

## [1.1.7] - 2026-04-17

### 🔧 核心算法改进

- **用温升速率积分计算温度**
  - 问题：实验数据 T=f(Q,I) 是从 25℃开始测的，段 2 从 17℃开始直接用插值会跳变
  - 修复：从实验数据推导温升速率 `dT/dt`，积分 `T(t+dt) = T(t) + dT/dt × dt`
  - 影响：温度从当前状态连续演化，支持任意起始温度

### 📐 物理意义

- 实验数据告诉我們"在这个容量和电流下，温度变化多快"
- 我们从当前温度开始，按这个速率演化
- 段 2 从 17℃开始，按实验数据的速率升温 → 连续！

## [1.1.8] - 2026-04-17

### 🐛 Bug 修复

- **修复电流标签检测（严格匹配 XA 格式）**
  - 问题：正则 `^(\d+)A?$` 匹配纯数字（25, 26, 27），温度值被误检为 25A, 26A, 27A
  - 修复：正则改为 `^(\d+)A$`（必须有 A 后缀）
  - 影响：正确检测 10 个电流（90A, 80A...10A）

### 📊 外推改善

- 之前：2.5A 从 90A 外推（36 倍）❌ 完全不可靠
- 现在：2.5A 从 10A 外推（4 倍）✓ 更可靠

### 📊 测试验证

**测试用例**：Reliance RS50 电芯（4.95 Ah）

**模拟条件**：
- 起始 SoC: 95%
- 恒功率：290W
- 时长：90 秒

**结果**：
- 电压：3.540V → 3.319V (Δ = 0.221V)
- 电流：81.91A → 87.39A (Δ = 5.48A)
- SoC: 95% → 52% (Δ = 43%)
- 放出容量：2.127 Ah

**平滑度**：
- 最大单步变化：0.0018V
- 平均单步变化：0.0002V
- ✓ 曲线平滑，无跳变
